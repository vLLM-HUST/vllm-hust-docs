# 甲方里程碑进度汇报材料

日期：2026-04-01

适用对象：

- 用于向甲方汇报当前第一个里程碑阶段进度
- 用于说明当前已完成工作、已具备基础、待补证据项和下一阶段推进重点

## 一、里程碑定位与阶段目标

当前第一个里程碑的定位，不是 9 个优化点的全面交付，而是课题公共基础能力的阶段性收口，重点完成以下四项工作：

1. 公共基线建立
2. 统一指标口径收敛
3. 统一评测与结果留痕方式确定
4. 阶段材料和证据链归口

基于这一定位，当前阶段的主要任务，是先把底层推理底座、统一接入入口、真实业务测试对象和基础观测能力贯通起来，为后续各优化方向持续提交结果建立统一基础。

---

## 二、阶段性总体判断

截至目前，项目总体实施方案与技术架构已经完成阶段性定型，`vllm-hust` 统一推理底座、`workstation` 本地控制面与 `EvoScientist` 首个真实多智能体测试对象三层关系已经明确，底层 serving 能力和部分上层接入能力已经具备。

当前阶段的重点，已经从“论证底层是否可行”转向“补齐真实场景验证、统一指标与画像 schema、跨硬件实测和流式数据闭环等证据链”。

建议采用的总收口表述如下：

“项目总体技术路线已完成阶段性定型，底层 serving 能力、统一接入入口和首个真实业务测试对象已形成贯通，当前工作重点转向真实场景验证、统一指标口径和跨硬件实测证据补齐。”

## 三、当前已完成的阶段性工作

### 1. 总体实施方案与技术架构完成阶段性定型

当前已经明确形成以下三层架构：

- `vllm-hust`：统一推理底座
- `vllm-hust-workstation`：应用平台，负责本地工作台、任务接入、指标聚合和演示控制
- `EvoScientist`：首个多智能体真实业务测试对象

同时，OpenAI-compatible 协议作为统一上层接入方式，plugin / platform 作为统一后端与异构硬件适配边界，整体技术路线已经清晰。

围绕这一架构，当前仓库分工也已经明确：

- `vllm-hust` 负责统一推理引擎能力
- `vllm-hust-workstation` 负责应用平台与本地集成入口
- `vllm-hust-benchmark` 负责评测、对比分析和结果沉淀
- `vllm-ascend-hust` 负责国产算力路径上的适配与优化
- `vllm-hust-docs` 负责实施方案、阶段材料和证据链归档

### 2. 底层 serving 能力已具备基础可用性

现有系统已经具备以下基础能力：

- 多模态基础处理能力
- 流式输出能力
- reasoning 输出能力
- 工具调用与结构化输出能力
- benchmark 与 metrics 基础采集入口
- 本地工作站与 Kubernetes / Helm 两类部署入口
- 独立 benchmark 仓库支撑的 baseline 与对比分析入口

### 3. 首个真实多智能体业务对象已接入

当前已经完成：

- `EvoScientist` 作为首个真实多智能体 workload 接入本地 `vllm-hust`
- `workstation -> EvoScientist -> vllm-hust` 默认桥接链路打通
- standalone `EvoScientist` 最小 smoke test 跑通

这表明项目已经从单纯架构讨论进入到真实任务对象验证阶段。

### 4. 统一后端与环境封装需求分析已完成

当前已经明确：

- 上层通过 OpenAI-compatible 接口统一接入
- 中层围绕 streaming、tool calling、structured outputs、reasoning 等能力形成一致能力边界
- 底层通过 plugin / platform 承接异构硬件与国产算力适配

这意味着环境封装与统一后端需求分析已经完成，后续重点将转向持续接入和验证，而不是重新定义总架构。

### 5. 国产算力方向已形成明确瓶颈分析框架

当前已收敛出一组主要瓶颈方向：

- 前处理开销
- TTFT / ITL 拉扯
- KV 传输与缓存
- 动态 batch 与通信路径

这为后续国产硬件专项优化和跨硬件实测提供了明确分析框架。

### 6. 负载画像与流式交互已具备基础抓手

当前已经具备围绕以下指标开展基础采集与画像分析的条件：

- TTFT
- ITL
- TPOT
- token throughput
- error type
- reasoning length
- tool density

这意味着负载画像与流式交互不再停留在概念层面，已经具备基础采集抓手和真实 workload 入口。

## 六、下一阶段推进重点

下一阶段建议围绕以下四项继续推进：

### 1. 补齐统一指标与统一基线

- 固化 TTFT / TPOT / 吞吐 / MFU / 单位 token 成本口径
- 固化基线评测流程、环境字段和结果归档格式

### 2. 补齐国产硬件首轮 baseline 数据

- 先完成至少 1 类国产芯片上的 7B-13B 基本链路跑通
- 形成首批可归档 baseline 数据和问题清单

### 3. 补齐真实 workload 验证和画像闭环

- 围绕 `EvoScientist` 形成更稳定的真实任务验证链路
- 补统一 schema、错误分类和阶段画像报表

### 4. 建立后续优化结果统一回收机制

- 各优化方向继续按既定分工推进
- 所有阶段结果按统一指标、统一格式和统一证据链回收到里程碑体系中

综合来看，当前第一个里程碑已经可以形成如下阶段性结论：

1. 总体实施方案与技术架构已经完成阶段性定型。
2. `vllm-hust` 统一推理底座、`workstation` 本地控制面与 `EvoScientist` 真实业务测试对象已经形成贯通。
